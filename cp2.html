FUNCTIONS IN C
Introduction
The core of structured programming is to divide a problem into smaller parts. Understanding the problem as a whole, then break into smaller, simpler and understandable parts, we call each of these parts of a program, a module. Definition: A function is a group of statements that together perform a task. The concept of functions is the way to implement the concept of modules. very C program has at least one function, which is main ().
A function in C is an independent module that will be called to do a specific task. The function main called by the Operating System, main in turn calls the other functions. When main is complete, the control returns to the Operating System.
A called function receives control from a calling function. When the called function completes its task, it returns the control to the calling function.
Purpose of a function is to receive zero or more pieces of data, operate on them and return at most one piece of data. A function can have side effects. Side effects can involve accepting data from outside the program, sending data out of the program or changing the value of the variable in the calling function.

Advantages of using functions:
1. Programs can be factored into understandable and manageable parts.
2. Functions provide a way to reuse code.

Introduction
There are basically two kinds of functions namely Built in functions and User defined functions.

Built in functions
Standard library functions are also known as built-in functions. Functions such as printf() , scanf() etc are standard library functions. These functions are already defined in header files (files with . h extensions are called header files such as stdio.

User defined functions:
C allows you to define functions according to your need. These functions are known as user-defined functions.
User Defined Functions must be both declared and defined.

Function Declaration and Function Definition.
Function Declaration : A function declaration tells the compiler about a function's name, return type, and parameters.
Function Definition: A function definition provides the actual body of the function.

Explanation for Function Declaration: A function declaration tells the compiler about a function name and how to call the function. A function declaration has the following parts − return_type function_name (parameter list );
Parameter names are not important in function declaration only their type is required, so the following is also a valid declaration
A function definition in C programming consists of a function header and a function body. Here are all the parts of a function − * Return Type − A function may return a value. The return_type is the data type of the value the function returns. Some functions perform the desired operations without returning a value. In this case, the return_type is the keyword void.
* Function Name − This is the actual name of the function. The function name and the parameter list together constitute the function signature.
* Parameters − A parameter is like a placeholder. When a function is invoked, you pass a value to the parameter. This value is referred to as actual parameter or argument. The parameter list refers to the type, order, and number of the parameters of a function. Parameters are optional; that is, a function may contain no parameters.
* Function Body − The function body contains a collection of statements that define what the function does.

Function Arguments
If a function is to use arguments, it must declare variables that accept the values of the arguments. These variables are called the formal parameters of the function.
*Formal parameters behave like other local variables inside the function and are created upon entry into the function and destroyed upon exit.
*Actual parameters are used to pass the value to the function from the calling function. While calling the function, the parameters used are called actual parameters.
While calling a function, there are two ways in which arguments can be passed to a function –
Call by value and Call by Reference ( Can be told that we are looking into examples of call by value now and that call by reference will be spoken about while teaching pointers).
