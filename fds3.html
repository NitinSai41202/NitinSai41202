QUEUE
Introduction to Queue
A Queue is a linear structure which follows a particular order in which the operations are performed.
The order is First In First Out (FIFO).
A good example of a queue is any queue of consumers for a resource where the consumer that came first is served first.
The difference between stacks and queues is in removing. In a stack we remove the item the most recently added; in a queue, we remove the item the least recently added.

Operations performed in the Queue
Insertion and Deletion
ENQUEUE:- The process of inserting an element in the queue.
DEQUEUE:- The Process of Deleting an element from the queue.
ENQUEUE will be done at REAR end
DEQUEUE will be done at FRONT end

Conditions to be followed
Overflow:- Attempt to insert an element , when the queue is full.
Underflow:- Attempt to delete an element, when the queue is empty.
Unlike stack, Queue has two ends,
1) FRONT – points to starting element
2) REAR – Points to the Last element.

Types of Queue
  1. Linear Queue
  2. Priority Queue
  3. Circular Queue
  4. Doubly ended Queue

The Priority Queue ADT
A priority Queue is an Extended version of the basic queue with the exception that a priority p must be assigned to each item at the time it is enqueued.
1. A priority queue is a queue in which each item is assigned a priority and items with a higher priority are removed before those with a lower priority, irrespective of when they are added.
2. Integer values are used for the priorities with a smaller integer value having a higher priority.
There are two basic types of priority queues.They are Bounded and Unbounded Priority queues.
1. A bounded priority queue restricts the priorities to the integer values between zero and predefined upper limit value.
2. An unbounded priority queue places no limits on the range of priorities.
APPLICATIONS OF PRIORITY QUEUE:
i. Prim's algorithm implementation can be done using priority queues.
ii. Dijkstra's shortest path algorithm implementation can be done using priority queues.
iii. In traffic light, depending upon the traffic, the colors will be given priority.

Circular Queue
1.In an Queue when elements are arranged in sequential manner but logically we assume it in circular format, then such queue is called as "Circular Queue".
2.It follows "First In First Out" method for insertion and deletion of elements.
3.Liner Queue may appear to be full although there may be space in the queue, this problem is overcome due to circular nature of circular queue.
4.A variable named "FRONT" stores position of the starting element of the queue. Its incremented on deletion of an element.
5.A variable named "REAR" stores position of the last element of the queue. Its incremented on Insertion of new element
6.To check Queue is Full or Not following condition is used front == (rear+1)%SIZE, if the condition is true then the Queue is full or else not full.
7.To check Queue is Empty or Not following condition is used rear == front, if the condition is true then the Queue is Empty or else not empty.

Double Ended Queue
1.A Queue in which inserting and deleting of elements is done from both the ends, such queue is called as Double Ended Queue(DeQueue).
2.Dequeue always consists of homogeneous list of elements.
3.There are two types of DeQueue. A. Input restricted dequeues B. Output restricted dequeues
4.Input restricted dequeues allows insertion only at one end but allows deletion of element from both the ends.
5.Output restricted dequeues allwos deletions of elements only at one end but allows insertion of element from both the ends.
